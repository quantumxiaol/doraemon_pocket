# 开发说明

本文档面向开发者，用于本地构建、运行和实机测试。

## 环境

项目配置为：

- Minecraft 1.20.1
- Fabric Loader
- Fabric API `0.92.9+1.20.1`
- Gradle/Loom 使用 JDK 21 运行
- Mod 字节码保持 Java 17 兼容

VS Code 已在本地 `.vscode/settings.json` 指向本机 JDK 21 路径。这个配置不会修改 shell 环境变量，并且 `.vscode/` 当前被 `.gitignore` 忽略，不会提交本机路径。

## 构建

使用项目自带的 Gradle Wrapper，不需要全局安装 Gradle：

```bash
JAVA_HOME="$(/usr/libexec/java_home -v 21)" ./gradlew build
```

这只是给单次命令指定 JDK 21，不会修改全局环境变量。

构建成功后，可用于实机测试的 Mod jar 位于：

```text
build/libs/doraemon_pocket-1.0.0.jar
```

不要使用：

```text
build/libs/doraemon_pocket-1.0.0-sources.jar
```

那个是源码包。

## 开发客户端

```bash
JAVA_HOME="$(/usr/libexec/java_home -v 21)" ./gradlew runClient
```

## 实机测试

把下面两个 jar 放进同一个 Minecraft `mods/` 文件夹：

```text
doraemon_pocket-1.0.0.jar
fabric-api-0.92.9+1.20.1.jar
```

建议重点测试：

- 创造模式物品栏是否出现 `Doraemon Pocket` 分组。
- 是否能拿到“竹蜻蜓”。
- 戴到头盔栏后，跳跃键是否上升。
- 戴到头盔栏后，潜行键是否下降。
- 空中按住前后左右移动键时，是否能按视角方向前进和转向。
- 在地面只按前后左右移动时，竹蜻蜓是否不会额外水平加速，也不会消耗耐久。
- 从高处离地或悬停时，竹蜻蜓是否会持续消耗耐久。
- 佩戴竹蜻蜓时，头顶旋翼是否会转动，离地后是否明显加快。
- 佩戴竹蜻蜓离地飞行时，是否能听到较低音量蜜蜂底声和幻翼拍风声混合出的旋翼音效。
- 高处落下是否不会受到摔落伤害。
- 生存模式主动飞行时耐久是否会消耗。
- 竹蜻蜓当前 6000 点耐久是否足够支持较长时间飞行测试。
- 创造模式物品栏是否能拿到“空气炮”。
- 空气炮右键是否播放音效并进入短暂冷却。
- 空气炮命中锥形范围内多个实体时，是否能造成少量伤害并击退。
- 空气炮近距离和远距离是否有明显伤害/击退衰减。
- 草、藤蔓等无碰撞植物是否不会阻挡气浪，实心方块是否会阻挡。
- 生存模式使用空气炮时耐久是否会消耗。
- 创造模式物品栏是否能拿到“休克枪”。
- 休克枪右键命中普通实体后，目标是否获得“休克”和虚弱效果，且移动明显被压制。
- 休克枪命中烈焰人等远程怪后，怪物是否约 5 秒持续丢失目标、停止导航，之后是否恢复。
- 休克枪命中后的怪物如果受到伤害，是否会立刻移除“休克”效果。
- 休克枪命中末影龙、凋灵、远古守卫者时，是否只触发短暂迟缓/虚弱，不进入完整休克。
- 创造模式物品栏是否能拿到“闪避斗篷”。
- 闪避斗篷装备在胸甲栏、主手、副手时是否都能触发闪避。
- 闪避斗篷对普通非环境伤害是否以约 32% 概率取消伤害。
- 闪避斗篷对箭、烈焰弹等投射物是否能稳定取消伤害，并尽量把投射物弹飞。
- 闪避斗篷对仙人掌、岩浆、浆果丛等环境接触伤害是否不会触发闪避。
- 闪避斗篷触发时是否消耗耐久。
- 闪避斗篷装备在胸甲栏时，背后是否能看到基础红色披风外观。
- 创造模式物品栏是否能拿到“时光包袱皮”。
- 一只手拿时光包袱皮、另一只手拿受损工具时，右键是否消耗 30 点经验并修满受损物品。
- 另一只手为空、物品无耐久或未受损时，时光包袱皮是否不会触发修复。
- 创造模式使用时光包袱皮是否不消耗经验。
- 创造模式物品栏是否能拿到“适应灯”。
- 适应灯在背包、主手或副手中时，是否持续获得“环境适应”效果。
- 适应灯效果下，溺水、岩浆/火焰、冰冻、仙人掌、虚空伤害是否被取消。
- 适应灯效果下，怪物攻击、玩家攻击、箭矢等实体伤害是否仍然生效。
- 创造模式物品栏是否能拿到“桃太郎饭团”。
- 桃太郎饭团右键敌对生物时，目标是否立刻放弃仇恨，并暂时不会伤害喂食玩家。
- 桃太郎饭团右键空气时，是否像雪球一样抛出并沿抛物线飞行，命中生物后触发安抚。
- 桃太郎饭团右键普通动物时，目标是否只在效果期间短距离跟随，时间到后恢复原版行为。
- 桃太郎饭团右键狼、猫、马等可驯服生物时，是否按原版归属逻辑永久驯服。
- 右键已生效的大体型目标时，是否可以直接骑乘并移动，且不需要鞍。
- 骑乘牛、马等临时坐骑时，是否可以正常上一格高的台阶。
- 潜行右键同一个已生效目标时，是否提前放归且不额外消耗饭团。
- 已属于其他玩家的驯服生物是否不会被桃太郎饭团抢夺。
- 创造模式物品栏是否能拿到“任意门”。
- 手持任意门执行 `/doraemon_pocket anywhere_door <x> <y> <z>` 时，是否把目标坐标写入这扇门。
- 任意门右键方块时，是否消耗手中任意门，并在点击处和目标坐标生成两端传送门。
- 任意门穿过一端时，是否能到达另一端，且不会立刻来回弹跳。
- 任意门普通右键门体时，是否像原版门一样切换开合状态。
- 任意门潜行右键任意一端时，是否关闭并回收到背包。
- 任意门破坏任意一片门时，另一端是否同步消失。
- 任意门在没有输入目的地、目的地空间不足或跨维度时，是否给出提示且不生成门。
- 创造模式物品栏是否能拿到“穿透环”。
- 穿透环右键墙面时，是否消耗手中穿透环，沿墙内侧最多扫描 96 格并连接到墙后的第一个空腔。
- 穿透环右键地面或天花板时，是否能按上下方向扫描空腔。
- 穿透环穿过一端时，是否能到达另一端，且不会立刻来回弹跳。
- 穿透环普通右键任意一端时，是否能到达另一端。
- 穿透环潜行右键任意一端时，是否回收到背包。
- 穿透环破坏任意一片环时，另一端是否同步消失。
- 穿透环对后方没有空腔的实心墙使用时，是否不会生成。
- 创造模式物品栏是否能拿到“石头帽”。
- 石头帽装备到头盔栏后，僵尸、苦力怕、末影人等是否不再主动追击玩家。
- 已经开始攻击玩家的怪物，在玩家戴上石头帽后是否会丢失目标。
- 戴着石头帽被怪物近战或怪物投射物命中时，是否不会受到伤害。
- 戴着石头帽靠近苦力怕时，苦力怕是否不会继续引爆。
- 石头帽装备到头盔栏后，头上是否能看到灰色海龟壳风格帽子。
- 戴着石头帽接触水体时，石头帽是否自动脱落并回到背包。
- 创造模式物品栏是否能拿到“名刀‘电光丸’”。
- 手持名刀“电光丸”时，正面飞来的箭、烈焰弹、三叉戟等是否能被自动弹开。
- 手持名刀“电光丸”被 3 格内怪物近战命中时，是否自动挥剑反击并取消本次伤害。
- 手持名刀“电光丸”靠近已经开始引信的苦力怕时，是否自动挥剑打断并击退。
- 名刀“电光丸”每次自动格挡或反击时是否明显消耗耐久。
- 名刀“电光丸”有电状态是否拥有明显更长的 640 点自动功能耐久。
- 名刀“电光丸”自动功能耐久耗尽时，是否变成“没电的电光丸”。
- 没电的电光丸是否不会自动格挡/反击，但仍可作为普通剑攻击。
- 创造模式物品栏是否能拿到“铜锣烧”。
- 铜锣烧是否可以食用。
- 小麦、牛奶、可可豆、糖、鸡蛋是否可以合成 4 个铜锣烧。

## 稳定性与性能备注

- 竹蜻蜓客户端只在输入变化或 10 tick 心跳时发送控制包；服务端会校验输入范围，控制包 30 tick 过期，并在断线或服务器停止时清理缓存。
- 休克枪不再直接写实体 `NoAI`，改用自定义“休克”状态效果逐 tick 压制移动、停止导航和清目标，受伤时移除该效果。
- 桃太郎饭团不会对普通生物写入永久 owner/tame 字段；普通生物只用内存里的临时服从状态，服务器停止时会释放已加载实体状态。
- 石头帽、电光丸、适应灯、闪避斗篷的常驻逻辑已做分频或全局定时清理，避免每玩家每 tick 重复做不必要扫描。
- 当前仍是玩法 MVP；如果要上长期多人服务器，后续还应继续做配置项、实体标签兼容、服务端压测和更细的权限/冷却策略。

## 音效策略

第一阶段优先使用 Minecraft 原版 `SoundEvents` 占位，不直接截取动画音频。这样可以先验证玩法，也避免引入额外授权风险。竹蜻蜓当前在客户端混合使用 `ENTITY_BEE_LOOP` 和 `ENTITY_PHANTOM_FLAP`，不额外打包音频文件。空气炮当前使用 `ITEM_FIRECHARGE_USE`。

休克枪当前使用 `ENTITY_LIGHTNING_BOLT_IMPACT`，闪避斗篷当前使用 `ITEM_SHIELD_BLOCK`。

时光包袱皮当前使用 `BLOCK_ANVIL_USE`。适应灯当前不播放循环音效，只通过状态效果体现。
